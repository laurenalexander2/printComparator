Print Comparator

Compare two digitized impressions from the same intaglio plate (etching, engraving, drypoint, mezzotint, and related processes). The tool reduces digitization variability, registers impressions into a shared coordinate space, and provides visual outputs designed for interpretation.

How it works

  1. Preprocess for comparability

    Convert to grayscale, standardize scale, and normalize contrast to reduce exposure differences, paper tone, and capture artifacts—so the comparison is driven more by print variation than digitization quirks.

  2. Align one impression to the other

    Detect shared landmarks, estimate the transform, and warp the target impression into the base impression’s coordinate space. If feature matching is weak or unstable, the alignment falls back to a gentler global method.

  3. Visual analysis (two modes)

    Mode 1 Absolute tonal difference

    A pixel-level absolute difference map in grayscale intensity. Best when changes are strictly additive (added tone, heavier inking, strengthened marks), because magnitude reads cleanly.

    Mode 2 Color overlay (overlaps neutralized)

    Tint each impression (red vs cyan) so regions that agree collapse toward neutral/darker values, while differences remain strongly colored. Best for rapid scanning and localizing change—especially when differences are mixed.

  4. Interactive controls

    Adjust thresholds and visibility, invert views, and switch comparison direction to tune sensitivity for tone versus line-level change.

The Tech Stack

Python-based image processing with an interactive Streamlit UI. The core idea: reliable registration + interpretable visual comparisons.

OpenCV (cv2)

Preprocessing (grayscale, resize, blur, histogram equalization), feature-based alignment (AKAZE + BFMatcher + Lowe ratio test), robust transform estimation (RANSAC homography), warping/registration (perspective + affine), difference representations (absdiff, normalization, thresholding, morphology, connected components), and overlay composition (channel merges, color conversions).

ECC affine fallback

When homography estimation fails validation or inputs are low-feature, the tool uses OpenCV’s ECC optimization to find a stable affine alignment (rotation/scale/translation). This avoids catastrophic “degenerate” warps and degrades gracefully.

NumPy

Vectorized pixel arithmetic for masks and overlays, fast threshold logic, feature vectors from change masks (grid ratios + global ratios), and general array manipulation.

Streamlit

Browser-based UI: file upload, sliders/toggles, session state, and real-time visualization of aligned outputs and analysis views.

scikit-learn (KMeans, optional)

Experimental clustering of impressions using feature vectors derived from change masks to explore hypothesized grouping/state structure.

Alignment safeguards

Homographies are validated (finite values, sane determinant/scale, sufficient inliers) before warping. If the transform looks unstable, the tool falls back to ECC affine alignment.

Line-focused alternatives

Optional strategies emphasize marks over paper tone: “ink-only” differences (adaptive thresholding on dark pixels) and/or edge-based comparisons (blur + Canny + XOR), with morphology and component filtering to suppress speckle.
